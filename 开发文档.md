# 记账本 — 开发文档

## 一、项目概述

记账本是一个纯前端本地记账 Web 应用，无需后端服务器，所有数据存储在浏览器 `localStorage` 中。支持收支记录、分类管理、图表统计、浅色/深色主题切换、数据导出/导入备份。

### 技术栈

| 技术 | 说明 |
|------|------|
| HTML5 | 单页应用，三个 section + 三个弹窗 |
| CSS3 | CSS 变量主题系统，奶油黄暖色调 |
| 原生 JavaScript | ES6+，IIFE 模块模式，无框架依赖 |
| Chart.js 4.4.7 | CDN 引入，饼图 + 折线图 |
| localStorage | 浏览器本地存储，持久化数据 |

### 运行方式

直接用浏览器打开 `index.html` 即可，无需构建、无需服务器。

```
jizhang/
├── index.html          # 唯一页面（三个 section + 三个弹窗 + 底部导航）
├── sm.md               # 需求文档（功能清单）

│   └── style.css       # 全部样式（含浅色/深色主题变量）

    ├── stats.js         # 统计模块：汇总卡片 + 饼图 + 折线图
```

## 三、页面结构

### 3.1 单页三区

应用只有一个 HTML 页面，通过底部导航切换三个 section：

| 页面 | section ID | 功能 |
|------|-----------|------|
| 记账 | `#page-record` | 支出/收入录入 + 最近 10 条记录 |
| 统计 | `#page-stats` | 汇总卡片 + 饼图 + 折线图 |
| 设置 | `#page-settings` | 主题切换 + 分类管理 + 数据备份 |

### 3.2 弹窗

| 弹窗 | ID | 用途 |
|------|-----|------|
| 编辑记录 | `#modal-edit` | 修改已有记录的金额/分类/日期/备注 |
| 分类管理 | `#modal-category` | 添加或编辑分类（名称/图标/颜色） |
| 确认删除 | `#modal-confirm` | 通用确认弹窗 |

### 3.3 底部导航

悬浮胶囊形导航栏，三个按钮（记账 ✏️ / 统计 📊 / 设置 ⚙️），点击切换页面并更新顶部标题。

## 五、模块详解

### 5.1 Storage 模块（storage.js）

**职责**：封装 localStorage 读写，管理记录和设置的 CRUD。

**存储键名**：
- `jizhang_records` — 记录数组
- `jizhang_settings` — 设置对象（主题 + 分类列表）

**数据结构 — 记录**：
```json
{
  "id": "1719235200000",
  "type": "expense",
  "amount": 35.5,
  "category": "餐饮",
  "date": "2026-06-24T15:30",
  "note": "午饭"
}
```

**数据结构 — 设置**：
```json
{
  "theme": "light",
  "expenseCategories": [
    { "name": "餐饮", "color": "#E86B4A", "icon": "🍜" }
  ],
  "incomeCategories": [
    { "name": "工资", "color": "#4CAF6A", "icon": "💰" }
  ]
}
```

**API 列表**：

| 方法 | 参数 | 说明 |
|------|------|------|
| `getRecords()` | — | 返回全部记录数组 |
| `addRecord(record)` | record 对象 | 添加记录，自动生成 `id`（时间戳），插入数组头部 |
| `updateRecord(id, updates)` | id, 部分字段 | 按 id 更新记录 |
| `deleteRecord(id)` | id | 按 id 删除记录 |
| `getSettings()` | — | 返回设置对象，自动升级旧数据 |
| `saveSettings(settings)` | 设置对象 | 保存设置 |
| `getCategories(type)` | `'expense'` / `'income'` | 返回对应分类数组 |
| `addCategory(type, cat)` | type, `{name,color,icon}` | 添加分类 |
| `updateCategory(type, index, updates)` | type, 索引, 部分字段 | 更新分类 |
| `deleteCategory(type, index)` | type, 索引 | 删除分类 |
| `exportData()` | — | 导出全部数据（records + settings） |
| `importData(data)` | 导入对象 | 覆盖写入 records 和 settings |

**数据升级**：`upgradeCategories()` 在 `getSettings()` 时自动为旧数据补充 `icon` 字段，兼容无图标的历史数据。

---

### 5.2 Record 模块（record.js）

**职责**：记账页面的全部交互逻辑。

**核心状态**：
- `currentType` — 当前类型（`'expense'` / `'income'`）
- `selectedCategory` — 当前选中的分类名
- `editingId` — 正在编辑的记录 id

**主要函数**：

| 函数 | 说明 |
|------|------|
| `init()` | 绑定事件、初始化日期、渲染分类和最近记录 |
| `setDefaultDateTime()` | 将日期输入框设为当前日期时间（`YYYY-MM-DDThh:mm`） |
| `renderCategories()` | 渲染分类选择网格（emoji 图标 + 配色背景） |
| `saveRecord()` | 校验输入、保存记录、清空表单、刷新列表 |
| `renderRecent()` | 渲染最近 10 条记录（图标 + 日期 + 金额） |
| `openEditModal(id)` | 打开编辑弹窗并填充数据 |
| `saveEdit()` | 保存编辑结果 |
| `createRipple(e)` | 创建波纹动画 |
| `flashButton(btn)` | 触发保存成功闪光动画 |
| `formatRecordDate(dateStr)` | 格式化日期显示，兼容新旧格式 |
| `refresh()` | 重新渲染分类和最近记录（供其他模块调用） |

**日期兼容**：同时支持旧格式 `"2024-01-15"` 和新格式 `"2024-01-15T14:30"`。

---

### 5.3 Stats 模块（stats.js）

**职责**：统计页面的图表和数据计算。

**核心状态**：
- `currentRange` — 当前时间范围（`'today'` / `'week'` / `'month'` / `'year'`）
- `pieChart` / `lineChart` — Chart.js 实例引用

**主要函数**：

| 函数 | 说明 |
|------|------|
| `init()` | 绑定时间切换按钮、首次渲染 |
| `refresh()` | 过滤记录 → 更新汇总 + 饼图 + 折线图 |
| `getDateRange()` | 根据 `currentRange` 返回 `{start, end}` 日期范围 |
| `filterRecords()` | 按时间范围过滤全量记录 |
| `parseLocalDate(dateStr)` | 本地时区安全的日期解析（避免 UTC 偏移） |
| `updateSummary(records)` | 更新总支出/总收入/结余三个卡片 |
| `updatePieChart(records)` | 绘制支出分类占比饼图 |
| `updateLineChart(records)` | 绘制支出趋势折线图（周/月按天，年按月） |

**图表配色**：饼图边框、折线图线条、坐标轴网格均跟随主题切换（浅色/深色模式自动适配）。

---

### 5.4 Settings 模块（settings.js）

**职责**：设置页面的全部交互逻辑。

**主要函数**：

| 函数 | 说明 |
|------|------|
| `init()` | 绑定主题/分类/备份事件、应用已保存主题 |
| `applyTheme()` | 从 settings 读取主题并设置 `data-theme` 属性 |
| `renderCategoryList(type)` | 渲染支出/收入分类管理列表 |
| `openCategoryModal(type, index)` | 打开分类弹窗（添加/编辑） |
| `saveCategory()` | 保存分类（名称 + 图标 + 颜色） |
| `editCategory(type, index)` | 暴露给全局的编辑入口 |
| `removeCategory(type, index)` | 暴露给全局的删除入口 |
| `exportData()` | 导出 JSON 备份文件并触发下载 |
| `importData(e)` | 从 JSON 文件恢复数据 |

---

### 5.5 App 入口（app.js）

**职责**：应用启动入口，初始化所有模块，管理页面导航。

**启动流程**：
1. `DOMContentLoaded` 触发 → 调用 `App.init()`
2. 绑定底部导航点击事件
3. 依次初始化 `RecordModule` → `StatsModule` → `SettingsModule`

**页面切换** `switchPage(pageName)`：
- 切换 `.nav-btn.active` 高亮
- 切换 `.page.active` 显示
- 更新顶部标题文字
- 切换到统计页时自动刷新图表

|------|------|------|

| 居家 | 🏠 | `#6BBF8A` |
| 娱乐 | 🎮 | `#B47ED8` |
| 医疗 | 💊 | `#E88B6B` |
| 教育 | 📚 | `#7EB5D6` |
| 其他 | 📦 | `#A89B8C` |

**收入分类（4 个）**：

| 名称 | 图标 | 颜色 |
|------|------|------|
| 工资 | 💰 | `#4CAF6A` |
| 兼职 | 💼 | `#6BBF8A` |
| 红包 | 🧧 | `#E06060` |
| 其他 | 💵 | `#5B9BD5` |

### 6.2 分类数据结构

每个分类包含三个字段：
- `name` — 分类名称（最多 10 字）
- `color` — 显示颜色（hex 值，用于图标背景和图表配色）
- `icon` — emoji 图标（未设置时默认 🏷️）

### 6.3 用户自定义

用户可在设置页添加、编辑、删除分类，自定义名称/图标/颜色。删除默认分类后不会自动恢复。

- 推荐 Chrome / Edge / Safari 最新版

## 十、已知限制

1. **纯本地存储** — 数据仅在当前浏览器，清除浏览器数据会丢失记录（需定期导出备份）
2. **无跨设备同步** — 不同浏览器/设备间数据不互通
3. **无服务端** — 不支持多用户、云端备份
4. **大量数据性能** — localStorage 容量约 5MB，万条以上记录可能变慢
5. **Chart.js CDN** — 需联网加载图表库，离线时图表不可用
