# 记账本 — 开发文档

## 一、项目概述

记账本是一个纯前端本地记账 Web 应用，无需后端服务器，所有数据存储在浏览器 `localStorage` 中。支持收支记录、分类管理、图表统计、浅色/深色主题切换、数据导出/导入备份。

### 1.1 技术栈

| 技术 | 说明 |
|------|------|
| HTML5 | 单页应用，三个 section + 六个弹窗 |
| CSS3 | CSS 变量主题系统，奶油黄暖色调 |
| 原生 JavaScript | ES6+，IIFE 模块模式，无框架依赖 |
| Chart.js 4.4.7 | CDN 引入，饼图 + 折线图 |
| localStorage | 浏览器本地存储，持久化数据 |
| Capacitor 7.x | Android APK 打包（WebView 壳） |

### 1.2 运行方式

- **Web 版**：直接用浏览器打开 `index.html`，无需构建、无需服务器
- **Android 版**：`记账本.apk` 安装到手机，Android 7.0+

## 三、页面结构

### 3.1 单页三区

应用只有一个 HTML 页面，通过底部导航或左右滑动切换三个 section：

| 页面 | section ID | 功能 |
|------|-----------|------|
| 记账 | `#page-record` | 支出/收入录入 + 最近 10 条记录 |
| 统计 | `#page-stats` | 汇总卡片 + 饼图 + 折线图 |
| 设置 | `#page-settings` | 主题切换 + 分类管理 + 数据备份 + 关于 |

### 3.2 弹窗（共六个）

| 弹窗 | ID | 用途 |
|------|-----|------|
| 编辑记录 | `#modal-edit` | 修改已有记录的金额/分类/日期/备注 |
| 分类管理 | `#modal-category` | 添加或编辑分类（名称/图标/颜色） |
| 确认删除 | `#modal-confirm` | 自定义确认弹窗（替代浏览器 confirm） |
| 提示信息 | `#modal-alert` | 自定义提示弹窗（替代浏览器 alert） |
| 日期选择 | `#modal-date` | 自定义日期时间选择器 |
| — | — | 以上弹窗均使用奶油黄主题圆角矩形样式 |

### 3.3 底部导航

悬浮胶囊形导航栏（`border-radius: 28px`），三个按钮（记账 ✏️ / 统计 📊 / 设置 ⚙️），支持点击和左右滑动切换，切换时有淡入动画。

## 五、模块详解

### 5.1 Storage 模块（storage.js）

**职责**：封装 localStorage 读写，管理记录和设置的 CRUD。

**存储键名**：
- `jizhang_records` — 记录数组
- `jizhang_settings` — 设置对象（主题 + 分类列表）

**数据结构 — 记录**：
```json
{
  "id": "1719235200000",
  "type": "expense",
  "amount": 35.5,
  "category": "餐饮",
  "date": "2026-06-24T15:30",
  "note": "午饭"
}
```

**数据结构 — 设置**：
```json
{
  "theme": "light",
  "expenseCategories": [
    { "name": "餐饮", "color": "#E86B4A", "icon": "🍜" }
  ],
  "incomeCategories": [
    { "name": "工资", "color": "#4CAF6A", "icon": "💰" }
  ]
}
```

**API 列表**：

| 方法 | 参数 | 说明 |
|------|------|------|
| `getRecords()` | — | 返回全部记录数组 |
| `addRecord(record)` | record 对象 | 添加记录，自动生成 `id`（Date.now），插入数组头部 |
| `updateRecord(id, updates)` | id, 部分字段 | 按 id 更新记录 |
| `deleteRecord(id)` | id | 按 id 删除记录 |
| `getSettings()` | — | 返回设置对象，自动升级旧数据 |
| `saveSettings(settings)` | 设置对象 | 保存设置 |
| `getCategories(type)` | `'expense'` / `'income'` | 返回对应分类数组 |
| `addCategory(type, cat)` | type, `{name,color,icon}` | 添加分类 |
| `updateCategory(type, index, updates)` | type, 索引, 部分字段 | 更新分类 |
| `deleteCategory(type, index)` | type, 索引 | 删除分类 |
| `exportData()` | — | 导出全部数据（records + settings + exportDate） |
| `importData(data)` | 导入对象 | 覆盖写入 records 和 settings |

**数据升级**：`upgradeCategories()` 在 `getSettings()` 时自动为旧数据补充 `icon` 字段（默认 🏷️），兼容无图标的历史数据。

### 5.2 Record 模块（record.js）

**职责**：记账页面的全部交互逻辑。

**核心状态**：
- `currentType` — 当前类型（`'expense'` / `'income'`）
- `selectedCategory` — 当前选中的分类名
- `editingId` — 正在编辑的记录 id

**主要函数**：

| 函数 | 说明 |
|------|------|
| `init()` | 绑定事件、初始化日期、渲染分类和最近记录 |
| `setDefaultDateTime()` | 将隐藏 input 和日期显示框设为当前日期时间 |
| `renderCategories()` | 渲染分类选择网格（emoji 图标 + 配色背景） |
| `saveRecord()` | 校验输入、保存记录、清空表单、刷新列表、触发闪光动画 |
| `renderRecent()` | 渲染最近 10 条记录（图标 + 日期 + 金额），绑定编辑/删除事件 |
| `openEditModal(id)` | 打开编辑弹窗并填充数据，兼容旧日期格式 |
| `saveEdit()` | 保存编辑结果 |
| `createRipple(e)` | 在按钮点击位置创建波纹动画 |
| `flashButton(btn)` | 触发保存成功闪光动画（reflow 重播） |
| `formatRecordDate(dateStr)` | 格式化日期显示，兼容 `"2024-01-15"` 和 `"2024-01-15T14:30"` |
| `refresh()` | 重新渲染分类和最近记录（供其他模块调用） |

**删除流程**：点击删除按钮 → `showConfirm()` 自定义确认弹窗 → 确认后删除 + 刷新。

### 5.3 Stats 模块（stats.js）

**职责**：统计页面的图表和数据计算。

**核心状态**：
- `currentRange` — 当前时间范围（`'today'` / `'week'` / `'month'` / `'year'`）
- `pieChart` / `lineChart` — Chart.js 实例引用

**主要函数**：

| 函数 | 说明 |
|------|------|
| `init()` | 绑定时间切换按钮、首次渲染 |
| `refresh()` | 过滤记录 → 更新汇总 + 饼图 + 折线图 |
| `getDateRange()` | 根据 `currentRange` 返回 `{start, end}` 日期范围 |
| `filterRecords()` | 按时间范围过滤全量记录 |
| `parseLocalDate(dateStr)` | 本地时区安全的日期解析，兼容 `T` 分隔符 |
| `updateSummary(records)` | 更新总支出/总收入/结余三个卡片 |
| `updatePieChart(records)` | 绘制支出分类占比饼图（颜色取自分类设置） |
| `updateLineChart(records)` | 绘制支出趋势折线图（周/月按天，年按月） |

**图表配色**：折线图线条使用主题色 `#D4893A`，饼图边框、坐标轴网格、文字颜色均跟随浅色/深色主题自动切换。

### 5.4 Settings 模块（settings.js）

**职责**：设置页面的全部交互逻辑。

**主要函数**：

| 函数 | 说明 |
|------|------|
| `init()` | 绑定主题/分类/备份事件、应用已保存主题 |
| `applyTheme()` | 从 settings 读取主题并设置 `data-theme` 属性 |
| `renderCategories()` | 渲染支出 + 收入两个分类管理列表 |

| `openCategoryModal(type, index)` | 打开分类弹窗，填充名称/图标/颜色 |

| `editCategory(type, index)` | 暴露给全局的编辑入口（通过 onclick 调用） |
| `removeCategory(type, index)` | 暴露给全局的删除入口，使用 `showConfirm()` 确认 |

| `importData(e)` | 从 JSON 文件恢复数据，使用 `showConfirm()` 确认 |

**关于区域**：设置页底部显示应用图标、名称（记账本 v1.0.0）、作者（大康）、简介。

### 5.5 App 入口（app.js）

**职责**：应用启动入口，初始化所有模块，管理页面导航和全局弹窗。

**启动流程**：
1. `DOMContentLoaded` 触发 → 调用 `App.init()`
2. 绑定底部导航点击事件
3. 绑定滑动手势（touchstart / touchmove / touchend）
4. 依次初始化 `RecordModule` → `StatsModule` → `SettingsModule`

**页面切换** `switchPage(pageName)`：
- 旧页面瞬间隐藏（`classList.remove('active')`）
- 新页面淡入（`classList.add('fade-in')`，300ms opacity 动画）
- 更新顶部标题 + 导航高亮
- 切换到统计页时自动刷新图表
- 动画期间防抖（`_animating` 标志位）

**滑动手势**：
- 监听 `#app` 容器的 touch 事件
- 水平距离 > 60px、水平 > 垂直×1.5、时间 < 500ms 判定为有效滑动
- 左滑 → 下一页，右滑 → 上一页
- 弹窗打开时自动禁用

**全局弹窗函数**：

| 函数 | 返回值 | 说明 |
|------|--------|------|
| `showConfirm(message)` | `Promise<boolean>` | 自定义确认弹窗，点击遮罩等同取消 |
| `showAlert(message)` | `Promise<void>` | 自定义提示弹窗，点击确定或遮罩关闭 |

这两个函数通过 `window.showConfirm` / `window.showAlert` 暴露给全局，替代浏览器原生 `confirm()` 和 `alert()`，保证样式与应用主题一致。

### 5.6 DatePicker 组件（datepicker.js）

**职责**：完全自定义的日期时间选择器，替代浏览器原生 `datetime-local` 输入框。

**核心状态**：
- `viewYear` / `viewMonth` — 当前日历视图的年月
- `selectedDay` — 选中的日期
- `hour` / `minute` — 选中的时间
- `targetInputId` — 关联的隐藏 input ID

**主要函数**：

| 函数 | 说明 |
|------|------|
| `init()` | 绑定所有日期显示框的点击事件和弹窗内按钮事件 |
| `open(inputId, cb)` | 打开选择器，读取已有值或默认当前时间 |
| `close()` | 关闭选择器 |
| `renderCalendar(animate)` | 渲染日历网格（7列×6行），支持淡入动画 |
| `updateTimeDisplay()` | 更新时/分显示 |
| `formatValue()` | 返回 `"YYYY-MM-DDThh:mm"` 格式字符串 |
| `formatDisplay(val)` | 返回 `"2026年6月24日 周二 15:30"` 可读格式 |

**日历网格逻辑**：
- 周一起始（一 二 三 四 五 六 日）
- 上月/下月日期灰色补位，补满 6 行（42 格）
- 今天显示琥珀色边框，选中日期为琥珀色实心圆
- 月份切换带淡入动画（220ms）

**时间调节**：−/+ 按钮逐小时/分钟增减，循环 00~23 / 00~59。

**使用方式**：HTML 中放置 `.date-display` 显示框 + `type="hidden"` 隐藏 input，通过 `data-target` 关联。点击显示框自动打开选择器，确定后同时更新隐藏值和显示文字。

## 八、Android 打包

### 8.1 打包方案

使用 [Capacitor](https://capacitorjs.com/) 将 Web 应用包装为 Android WebView 壳。

### 8.2 环境要求

| 依赖 | 版本 |
|------|------|
| Node.js | 18+ |
| Android Studio | 含 SDK 36 |
| Java | Android Studio 自带 JBR 21 |
| Gradle | 8.14.3（自动下载） |

### 8.3 打包步骤

```bash
# 1. 安装 Capacitor
npm init -y
npm install @capacitor/core @capacitor/cli @capacitor/android

# 2. 创建 www 目录，复制 Web 资源
mkdir www
cp index.html www/
cp -r css www/
cp -r js www/
cp 封面.png www/

# 3. 初始化 Capacitor
npx cap init "记账本" com.jizhang.app --web-dir www

# 4. 添加 Android 平台
npx cap add android
npx cap sync android

# 5. 设置环境变量
export JAVA_HOME="E:/Android/Android Studio/jbr"
export ANDROID_HOME="E:/Android/Android SDK"

# 6. 构建 APK
cd android
./gradlew assembleDebug

# 输出: android/app/build/outputs/apk/debug/app-debug.apk
```

### 8.4 构建注意事项

| 问题 | 解决方案 |
|------|---------|
| Gradle 下载超时 | `gradle-wrapper.properties` 改用腾讯镜像 |
| Maven 依赖下载慢 | `build.gradle` 添加阿里云 Maven 镜像 |
| 中文路径报错 | `gradle.properties` 添加 `android.overridePathCheck=true` |
| `webDir` 不合法 | 不可用 `"."`，需指向子目录如 `"www"` |

## 十一、已知限制

1. **纯本地存储** — 数据仅在当前浏览器，清除浏览器数据会丢失记录（需定期导出备份）
2. **无跨设备同步** — 不同浏览器/设备间数据不互通
3. **无服务端** — 不支持多用户、云端备份
4. **大量数据性能** — localStorage 容量约 5MB，万条以上记录可能变慢
5. **Chart.js CDN** — 需联网加载图表库，离线时图表不可用
6. **Android WebView** — 部分 CSS 特性在旧版 WebView 中可能不完全支持
